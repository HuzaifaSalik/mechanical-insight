# Deployment Guide - Company Insight Website

This guide will walk you through deploying your Flask website to Render.

## Prerequisites

1. A [Render account](https://render.com) (free to create)
2. A [GitHub](https://github.com) or [GitLab](https://gitlab.com) account
3. Your code pushed to a Git repository
4. A Gmail account for sending emails (or other SMTP provider)

## Step 1: Prepare Your Repository

1. **Initialize Git (if not already done)**
   ```bash
   git init
   git add .
   git commit -m "Initial commit - Company Insight website"
   ```

2. **Push to GitHub/GitLab**
   - Create a new repository on GitHub or GitLab
   - Follow their instructions to push your code

   ```bash
   git remote add origin https://github.com/your-username/company-insight.git
   git branch -M main
   git push -u origin main
   ```

## Step 2: Set Up Gmail App Password

To send emails from your website, you need a Gmail App Password:

1. Go to your [Google Account Security](https://myaccount.google.com/security)
2. Enable **2-Factor Authentication** if not already enabled
3. Go to [App Passwords](https://myaccount.google.com/apppasswords)
4. Select "Mail" and generate a password
5. **Save this password** - you'll need it for Render environment variables

## Step 3: Set Up Google reCAPTCHA (Optional but Recommended)

Protect your contact form from spam:

1. Go to [Google reCAPTCHA Admin](https://www.google.com/recaptcha/admin)
2. Click **Create** and fill in:
   - Label: "Company Insight"
   - reCAPTCHA type: reCAPTCHA v2 (Checkbox)
   - Domains: Add your Render domain (e.g., `company-insight.onrender.com`)
3. Accept terms and submit
4. **Copy the Site Key and Secret Key** - you'll need these

## Step 4: Deploy to Render

### Option A: Using Blueprint (Recommended - Automatic Setup)

1. **Log in to [Render Dashboard](https://dashboard.render.com)**

2. **Click "New +" → "Blueprint"**

3. **Connect your repository**
   - Select GitHub or GitLab
   - Choose your repository
   - Grant access

4. **Render will detect `render.yaml`**
   - Service name: `company-insight`
   - Database: `company-insight-db` (PostgreSQL)

5. **Set Environment Variables**

   Click on your web service and add these environment variables:

   | Variable | Value | Notes |
   |----------|-------|-------|
   | `FLASK_ENV` | `production` | Auto-set |
   | `SECRET_KEY` | (auto-generated) | Auto-generated by Render |
   | `DATABASE_URL` | (auto-connected) | Auto-connected to PostgreSQL |
   | `MAIL_SERVER` | `smtp.gmail.com` | Gmail SMTP server |
   | `MAIL_PORT` | `587` | Auto-set |
   | `MAIL_USE_TLS` | `True` | Auto-set |
   | `MAIL_USERNAME` | `your-email@gmail.com` | Your Gmail address |
   | `MAIL_PASSWORD` | `xxxx xxxx xxxx xxxx` | Gmail App Password (16 chars) |
   | `MAIL_DEFAULT_SENDER` | `noreply@companyinsight.com` | From email address |
   | `RECAPTCHA_PUBLIC_KEY` | `6Lc...` | From Google reCAPTCHA (optional) |
   | `RECAPTCHA_PRIVATE_KEY` | `6Lc...` | From Google reCAPTCHA (optional) |

6. **Deploy**
   - Click "Apply" to start deployment
   - Wait 5-10 minutes for first deployment

### Option B: Manual Setup

If you prefer manual setup or the Blueprint doesn't work:

1. **Create PostgreSQL Database**
   - Go to Render Dashboard
   - Click "New +" → "PostgreSQL"
   - Name: `company-insight-db`
   - Plan: Free
   - Click "Create Database"
   - **Copy the Internal Database URL**

2. **Create Web Service**
   - Click "New +" → "Web Service"
   - Connect your repository
   - Configure:
     - Name: `company-insight`
     - Environment: `Python 3`
     - Build Command: `pip install -r requirements.txt && flask db upgrade`
     - Start Command: `gunicorn run:app`
     - Plan: Free

3. **Add Environment Variables** (same as Option A above)

4. **Deploy**

## Step 5: Initialize the Database

After your first deployment succeeds:

1. **Go to your web service on Render**

2. **Click "Shell" tab** (opens a terminal to your app)

3. **Run database migrations**
   ```bash
   flask db upgrade
   ```

4. **Seed services**
   ```bash
   flask seed_services
   ```

5. **Create admin user**
   ```bash
   flask create_admin
   ```
   - Follow the prompts to create your admin account

## Step 6: Access Your Website

1. Your website will be live at: `https://company-insight.onrender.com` (or your custom name)

2. **Test the site:**
   - Homepage: `https://your-app.onrender.com/`
   - Admin panel: `https://your-app.onrender.com/admin/login`

3. **First-time admin login:**
   - Use the credentials you created in Step 5

## Step 7: Configure Custom Domain (Optional)

1. **In Render Dashboard:**
   - Go to your web service
   - Click "Settings" → "Custom Domains"
   - Click "Add Custom Domain"

2. **Add your domain:**
   - Enter your domain (e.g., `www.companyinsight.com`)
   - Follow DNS configuration instructions

3. **Update DNS records:**
   - Add CNAME or A record as instructed by Render
   - Wait for DNS propagation (can take up to 48 hours)

4. **SSL Certificate:**
   - Render automatically provisions Let's Encrypt SSL
   - Your site will be HTTPS-enabled

## Important Notes

### Free Tier Limitations

- **Web Service:**
  - Sleeps after 15 minutes of inactivity
  - First request after sleep takes ~30 seconds
  - 750 hours/month of runtime

- **Database:**
  - 1GB storage
  - Expires after 90 days (free tier)
  - Automatic backups not included

### Performance Tips

1. **Prevent sleeping (Paid plans only):**
   - Upgrade to paid plan ($7/month)
   - Or use a service like UptimeRobot to ping your site

2. **Database persistence:**
   - For production, consider upgrading to paid PostgreSQL ($7/month)
   - Free tier database expires after 90 days

3. **Static files:**
   - Consider using a CDN for images and static assets
   - Cloudflare can provide free CDN services

## Troubleshooting

### Build Fails

- Check build logs in Render Dashboard
- Ensure `requirements.txt` is complete
- Verify Python version compatibility

### Database Connection Error

- Verify `DATABASE_URL` is set correctly
- Check PostgreSQL database is running
- Run `flask db upgrade` in Shell

### Emails Not Sending

- Verify Gmail App Password (not regular password)
- Check 2FA is enabled on Gmail
- Ensure `MAIL_USERNAME` and `MAIL_PASSWORD` are correct
- Test with a simple email first

### Site is Slow

- Free tier apps sleep after inactivity
- First request after sleep takes ~30 seconds
- Consider paid plan for always-on service

### Static Files Not Loading

- Check file paths are correct
- Ensure static files are committed to Git
- Clear browser cache

## Monitoring & Maintenance

1. **View Logs:**
   - Go to Render Dashboard → Your Service → Logs
   - Monitor for errors and performance issues

2. **Set up Notifications:**
   - Render → Service → Settings → Deploy Notifications
   - Get email/Slack alerts for deployments

3. **Backups:**
   - Regularly backup your database
   - Use Render's manual backup feature
   - Or set up automated backups with paid plan

## Updating Your Site

When you make changes to your code:

1. **Commit and push to Git:**
   ```bash
   git add .
   git commit -m "Description of changes"
   git push
   ```

2. **Automatic deployment:**
   - Render automatically deploys when you push to your branch
   - Monitor progress in Render Dashboard

3. **Manual deployment:**
   - Go to Render Dashboard → Your Service
   - Click "Manual Deploy" → "Deploy latest commit"
